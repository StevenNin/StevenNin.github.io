了解vuex 主要是理解vue 的之间的饿数据传递

  • 父组件A 向 子组件B 传递数据 通过props的方法 【props是只读的属性】
  • 子组件B 向 父组件A 发送数据 通过emit

或者event bus 统一分发处理数据

vuex 的理念： 一个数据仓库，所有组件可以共享，

理解数据流

state - > view >actions > state

驱动应用的数据源 》以声明的方式将state映射到视图上》view上用户的操作经过后响应后影响状态

组件越来越多的话就会影响性能，所以就把组件的共享状态抽离出来，全局单例模式进行管理，组成一个巨大的识图，不论在树的任何节点，任何组件都可以获取状态和触发行为

可以从国家的法律发布来理解vuex ，法律人人可以用，可以看getter， 但是改变法律需要提出申请

修改一条法律的时候就用mutation，修改法律的一个章节的话就用actions 。如果要重新写法律的话

就需要module ，把法律细分每一个章节的部分都会有其独立的功能团队来完成编写

简单单一的提交用mutation 但是复杂的就可以用actions

注意

  1. 所有数据在state中一开始就要声明写好，初始化，后面需要增加新数据的话，就需要使用vue ，set
  2. 使用常量代替mutation 时间类型
  3. mutation 必须事同步函数
  4. mutation 的操作可以拿过来import后转成组件里的computed中当作方法来使用，因为state和getters 都是查看数据，mutation可以修改数据。
  5. action 是提交一次mutaion（可以从下面的代码中看出对比），而不是如mutation直接修改数据。

Mutation 实际提交的动作如果，想要提交参额外的数的话，在 mutaition中的commit 中传参， 即是mutation 的载荷 （payload）

action怎么触发mutation的, 和怎么触发一个action

getters像组件中的computed ，但是getters 是操作station的，对原始的数据进行进一步的加工，如果页面都个模块，把关键数据都放在getters 里的话 ，直接子啊业务组件中就可以获取了，很方便
