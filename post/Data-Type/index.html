  1. 简单数据类型（除object以为的数据类型）。直接存入stack
  2. 复杂值存heap地址。存入stack。object 变量的读取就是对象的引用

数据的存储的方式不同 ，导致对象的数据的应用汇出爱你的一些问题，

一些被创建后但是后期未被引用的数据就会被 ，内存回收的时候 ie6有bug，正常的页面onclick的缓存后的垃圾不进行回收处理

浅拷贝与深拷贝的理解和处理

浅拷贝至拷贝，拷贝数据地址（数据的引用），但是不拷贝数据 本身的值， 也不在新的内存区新建数据，

深拷贝：拷贝数值数据后，然后在新的内存区复制数据，相同的数据但是不存在同一个地址，有着不同的数据存储区

垃圾回收机制

什么是垃圾，就不会再被用到的数据，浏览器用可达数据统称，因为浏览器回收垃圾数据的时候，会从根节点开始遍历所有对象，回收不可达值所占据的内存

回收机制的遍历会对数据的引用添加次数，当引用次数为0 的时候，就可以对数据进行拉回
